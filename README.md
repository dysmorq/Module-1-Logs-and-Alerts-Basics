# Module-1-Logs-and-Alerts-Basics
The goal here is to understand what logs are, why they matter and how to spot suspicious or abnormal activities.

## 1.1: What is a log

A log is record of events generated by computer systems, applications or network devices.

A SOC analyst detects threats by noticing abnormal patterns in the logs.

## 1.2: Common log types for a SOC

- **System logs** – OS events, e.g., Windows Event Viewer, Linux `journalctl`.
- **Application logs** – Web servers, databases, email systems.
- **Security logs** – Firewall logs, antivirus alerts, intrusion detection system alerts.
- **Network logs** – Traffic flow, firewall logs, router events.

SOC analysts use these logs to monitor the activities from computer systems, applications or network devices and check for unusual behaviour. The analyst will then investigate any noted suspicious activity, document & escalate confirmed threats.
**Log Type	  Source	                Example Event	                    Why It Matters**
  System	    Linux -journalctl	      Failed SSH login	                Could indicate brute-force attack
  Security	  Windows Event Viewer	  User added to admin group	        Could be unauthorized privilege escalation
  Network	    Router logs	            Traffic spike to unknown IP	      Could be data exfiltration
